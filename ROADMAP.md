## 二、繼承自國小版 v3.10.0 的功能

以下功能完全沿用（程式碼相同），未來如有更新會與國小版同步：

### 核心架構
- 純前端單一 HTML、PWA 安裝、Service Worker 離線快取、localStorage 儲存設定、響應式設計（手機/平板/桌機）

### PDF 擷取
- 拖拉/點擊上傳、PDF.js v4.4.168 懶載入、第一頁縮圖預覽 + 全屏 lightbox、字數過長警告

### 審查項次資料
- 46+ 項審查標準內嵌（國中版專屬）、可編輯提示詞、項次即時搜尋、項次白話說明、特殊班級警告

### AI 整合
- Google Gemini API（gemini-2.5-flash / 2.5-pro / 2.0-flash）、OpenAI API、自動重試（3 次）、友善中文錯誤訊息

### 審查流程
- 單項審查（一次審一個項次）、批次審查（同時排程多個項次，含進度、暫停、續跑）、自動偵測通過/待修正、結果分組卡片呈現

### 結果呈現與分享
- Markdown / 純文字複製、列印、AI 生成修正稿、Web Share API 分享、LINE 分享、截圖下載（html2canvas）、Email 寄送主任、Word 報告下載（.doc）

### 免 API 金鑰 UX
- 三步驟引導（複製提示詞 → 開 Gemini → 貼回結果）、可收合/展開引導面板

### UI 設計
- 三步驟流程配色（藍/紫/琥珀）、Hero Header 漸層 + 裝飾球、字型大小切換（小/中/大）、教學引導 Overlay、Canva 教學簡報入口

### 社群分享與品牌
- Open Graph / Twitter Card meta 標籤、PWA manifest 完整設定

> 📚 完整功能歷程請參考[國小版 ROADMAP](https://github.com/cagoooo/curriculum/blob/main/ROADMAP.md)。

## 三、國中版專屬未來規劃

### 🟢 容易做（短期）
- [ ] **JHS 教學駕駛艙簡報**：目前 Canva 連結 `canva.link/115curriculum` 為國小版教學簡報；可考慮製作國中專屬教學簡報並更新連結
- [ ] **教師授課節數 7-3 項次**：原國中端 0505 審查文件引用「國民小學教師每週授課節數實施要點」（疑為原文未替換為國中版本）；後續追蹤教育局是否發布國中專屬要點，發布後同步更新提示詞
- [ ] **localStorage 跨工具防呆**：`tyc_last_review` 在跨國中／國小兩端時可能顯示對方的項次（如國小用過 6-2-1-5 後切到國中會看到「五年級」項次），考慮 namespace 為 `tyc_jhs_*` 或在顯示前驗證項次存在
- [ ] **特殊班級項次（5-1-2 / 5-1-3 / 5-1-4 / 6-2-3）**：目前僅基本 prompt，建議跟教育局徵詢國中端特教/藝才/體育班的具體審查重點，補上完整提示詞

### 🟡 中期功能升級
- [ ] **9 年級畢業考後規劃（3-3）智慧檢核**：偵測規劃內容是否「延續學習」，過濾純休閒活動
- [ ] **跨學年版本對應檢核（5-3-1）**：自動驗證 8 年級表呈現 114-115 兩年版本、9 年級表呈現 113-115 三年版本
- [ ] **議題融入時數計算（3-2 / 6-3）**：自動加總性平 6 節/學期、家庭教育 6 節/學年等法定議題節數，超標/不足即提醒

### 🔴 長期目標
- [ ] **特殊類型班級（特教/藝才/體育）國中專屬審查模組**：依各班型課綱建立完整提示詞，目前僅有基本框架
- [ ] **與國小版共用 Component**：把共用 UI 與資料層拆成獨立 module，避免兩端各維護一份；同時保留兩端的 prompt 差異

## 四、技術負債（與國小版共通）

- LocalStorage 與 Service Worker 共用 `cagoooo.github.io` origin，但國中／國小已透過 scope (`/JHScurriculum/` vs `/curriculum/`) + cache name (`jhs-curriculum-v1.0.0` vs `curriculum-v3.x`) 完全隔離，不會互相覆蓋
- API 金鑰（`tyc_apikey`）在兩端共用 — 屬於有意設計（使用者只需設定一次），不視為負債
- 上次審查記錄（`tyc_last_review`）在兩端共用會造成顯示錯亂 — 已列入「容易做」短期改善
