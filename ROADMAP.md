## 一、版本歷程

### v1.1.0-jhs（2026-05-06）— 視覺整體重新設計（編輯排版風）

依使用者回饋，整體 UI 從原墨綠系（#1a8a7a）改為**編輯排版風 / Editorial**，提升閱讀舒適度與專業審查質感。**所有審查邏輯、提示詞、PDF 擷取與 AI 整合完全沿用 v1.0.1，無功能性變動。**

#### 設計系統重寫

| 元素 | v1.0.x | v1.1.0 |
|------|------|------|
| 主色 | 墨綠 #1a8a7a | **C2 Olive 橄欖 #6b8a5e**（土地系黃綠） |
| 副色 | 漸層墨綠/藍/紫/琥珀 | **Olive Deep #4a6342**（深橄欖）+ **Copper #c97a3b**（銅紅點綴） |
| 背景 | 淺薄荷 #eef6f5 | **奶油米色 #f8f5e8**（暖紙感 paper） |
| 卡片底色 | 純白 + 漸層裝飾條 | 純白 + 左上短橄欖色條 + 左側狀態色條 |
| 標題字 | Nunito + Noto Sans TC | **Source Serif 4 + Noto Serif TC**（襯線編輯體） |
| 等寬字 | — | **JetBrains Mono**（小標、版本、頁碼） |
| 按鈕 | 圓角矩形漸層 | **橄欖膠囊 pill** + 銅紅次要按鈕 |

#### Layout 變更

- **Hero Header → C-T 頂部 5 步驟條**：原本的大型漸層 Hero 改成 sticky 橄欖色頂部進度帶，左側放品牌（含義式銅紅斜體 "AI 審查工具"），右側 5 個圓點對應 `API 設定 → 上傳 PDF → 選擇項次 → 送出審查 → 閱讀結果`，會依操作即時亮起（done = ✓ 白底、active = 銅紅放大 + 白圈）
- **Sub-bar**：把 Canva 教學 CTA + PWA 安裝橫幅獨立到米色子帶,讓主進度條視覺乾淨
- **主畫面寬度**：940/1040px → **置中 1000px**
- **卡片標頭**：新增「STEP 01 · 共 03」JetBrains Mono 小標 + 圓形編號徽章（01/02/03）
- **AI 注意事項卡片**：黃色警告 → 編輯感「拉式引言」（米色底 + 銅紅左條 + `注意` mono tag）

#### 結果頁（R3 狀態色條風格）

- 結果卡片改為 **左側 8px 色條 + 白底**：通過綠 #2f7d5b、待補強琥珀 #b07a14、不通過紅 #a83f2b（土地系飽和度，避免螢光感）
- 修正建議區（fix-box）改成 mono `PROBLEM` 標籤 + 米黃色底
- Markdown 標題用 Source Serif 4 襯線 + 橄欖深綠 left-border

#### 動態進度條（前後端打通）

新增 `updateStepperProgress()`，掛在 `updateButtons()` / 送出審查 / 渲染結果 / `clearResult()` / 初始化 等所有狀態變動點，讓頂部 5 圓點隨使用者實際進度即時更新。

#### 其他細節

- `manifest.json`：theme_color 改 `#6b8a5e`、background_color 改 `#f8f5e8`（PWA 啟動畫面也跟新色系）
- `sw.js`：SW_VERSION 升 `v1.1.0-jhs`、CACHE_NAME 升 `jhs-curriculum-v1.1.0`（強制使用者拉新版）
- 響應式：880px 以下頂部步驟條塌成上下兩段、520px 以下標籤自動隱藏只剩圓點
- 列印樣式：header 改成簡潔橄欖底線 + 隱藏 stepper / sub-bar
- 浮動 Canva 按鈕改銅紅色（從紫色），與整體色系統一

## 二、繼承自國小版 v3.10.0 的功能

以下功能完全沿用（程式碼相同），未來如有更新會與國小版同步：

### 核心架構
- 純前端單一 HTML、PWA 安裝、Service Worker 離線快取、localStorage 儲存設定、響應式設計（手機/平板/桌機）

### PDF 擷取
- 拖拉/點擊上傳、PDF.js v4.4.168 懶載入、第一頁縮圖預覽 + 全屏 lightbox、字數過長警告

### 審查項次資料
- 46+ 項審查標準內嵌（國中版專屬）、可編輯提示詞、項次即時搜尋、項次白話說明、特殊班級警告

### AI 整合
- Google Gemini API（gemini-2.5-flash / 2.5-pro / 2.0-flash）、OpenAI API、自動重試（3 次）、友善中文錯誤訊息

### 審查流程
- 單項審查（一次審一個項次）、批次審查（同時排程多個項次，含進度、暫停、續跑）、自動偵測通過/待修正、結果分組卡片呈現

### 結果呈現與分享
- Markdown / 純文字複製、列印、AI 生成修正稿、Web Share API 分享、LINE 分享、截圖下載（html2canvas）、Email 寄送主任、Word 報告下載（.doc）

### 免 API 金鑰 UX
- 三步驟引導（複製提示詞 → 開 Gemini → 貼回結果）、可收合/展開引導面板

### UI 設計
- 三步驟流程配色（藍/紫/琥珀）、Hero Header 漸層 + 裝飾球、字型大小切換（小/中/大）、教學引導 Overlay、Canva 教學簡報入口

### 社群分享與品牌
- Open Graph / Twitter Card meta 標籤、PWA manifest 完整設定

> 📚 完整功能歷程請參考[國小版 ROADMAP](https://github.com/cagoooo/curriculum/blob/main/ROADMAP.md)。

## 四、技術負債（與國小版共通）

- LocalStorage 與 Service Worker 共用 `cagoooo.github.io` origin，但國中／國小已透過 scope (`/JHScurriculum/` vs `/curriculum/`) + cache name (`jhs-curriculum-v1.0.0` vs `curriculum-v3.x`) 完全隔離，不會互相覆蓋
- API 金鑰（`tyc_apikey`）在兩端共用 — 屬於有意設計（使用者只需設定一次），不視為負債
- 上次審查記錄（`tyc_last_review`）在兩端共用會造成顯示錯亂 — 已列入「容易做」短期改善
